# Integrate Dynamic Design Intelligence into Agent Backend

Replace the hardcoded PM design rules and developer component writing with data-driven design systems (via `ui-ux-pro-max`) and curated component fetching (via 21st.dev Magic MCP), so the pipeline produces diverse, high-quality UIs tailored to each business type.

## User Review Required

> [!IMPORTANT]
> **21st.dev Magic API Key** — You'll need a 21st.dev API key. Get one at https://21st.dev/magic/console and add it to `agent-backend/.env` as `TWENTYFIRST_API_KEY=<your-key>`.

> [!WARNING]
> **Cost Impact** — Each Magic API call is a network request to 21st.dev. With ~6-8 components per project and parallel generation, expect ~6-8 API calls per generation run. Confirm this is acceptable.

> [!IMPORTANT]
> **Python Dependency** — The `ui-ux-pro-max` design system generator requires Python 3. The agent-backend will `exec` the Python script as a subprocess. Confirm Python 3 is available in your deployment environment.

## Proposed Changes

### Component 1: Design System Generator

Wraps the `ui-ux-pro-max` Python script as a callable module. Runs **before** the PM LLM call to produce a data-driven design system (style, colors, typography, layout rules) based on the business type/industry.

#### [NEW] [designSystemGenerator.ts](file:///Users/idorabin/shadow_clone_antigravity/agent-backend/src/orchestrator/designSystemGenerator.ts)

- Export `generateDesignSystem(businessInput: BusinessInput): Promise<string>`
- Extracts query keywords from `businessInput` fields (businessName, description, tone, targetAudience, services)
- Runs `python3 <skills-path>/scripts/search.py "<keywords>" --design-system -f markdown` via `child_process.execAsync`
- Returns the raw markdown output (style, colors, typography, layout, anti-patterns)
- Has a **timeout** (15s) and **fallback**: if Python fails, returns a minimal hardcoded default so the pipeline doesn't crash
- Logs via the existing `log()` utility

### Component 3: 21st.dev Magic Component Tool

Wraps the 21st.dev Magic API as a LangChain tool that the developer LLM can call to fetch production-quality component templates.

#### [NEW] [magicComponentTool.ts](file:///Users/idorabin/shadow_clone_antigravity/agent-backend/src/orchestrator/magicComponentTool.ts)

- Export `createMagicTool()` factory function (mirrors the pattern in `devTools.ts`)
- Makes an HTTP POST to the 21st.dev Magic API with the component description
- Returns the component code as a string, or a graceful error message if the API is unavailable
- Includes a **timeout** (10s) and **retry** (1 retry on 429/500)
- Reads `TWENTYFIRST_API_KEY` from `process.env`
- If no API key is set, the tool returns a message saying "Magic component library not configured — write the component from scratch"
